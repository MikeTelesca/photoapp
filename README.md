# PhotoApp — AI Real Estate Photo Editor

AI-powered photo editing pipeline for real estate photographers. Upload bracketed photos via Dropbox, get back HDR-merged, color-corrected, professionally edited photos in minutes.

**Live:** https://photoapp-lilac.vercel.app

## What it does

1. Photographer pastes a Dropbox shared folder link (or uploads files directly)
2. App pulls all bracketed photos from Dropbox
3. Sends each bracket set to Google Gemini 3 Pro Image
4. AI does HDR merge + window pull + color correction + perspective fix in one pass
5. Admin reviews edits in a before/after gallery, approves/rejects/regenerates
6. Download approved photos as ZIP or save to Dropbox

## Stack

- **Frontend:** Next.js 14 App Router, TypeScript, Tailwind CSS
- **Auth:** NextAuth.js v5 (credentials-based)
- **Database:** Neon Postgres via Prisma
- **AI:** Google Gemini 3 Pro Image with intelligent fallback cascade
- **Storage:** Dropbox API (file ingestion + edited output delivery)
- **Hosting:** Vercel

## Architecture

### Database Schema

| Model | Purpose |
|-------|---------|
| `User` | Photographers and admins; role-based access; monthly AI cost tracking |
| `Job` | A property shoot session; stores preset, TV/sky styles, status, cost tracking |
| `Photo` | Output image; one per bracket group; tracks status, EXIF, error attempts |
| `Preset` | Editing style template (e.g., "Luxury", "Bright & Airy") with prompt modifiers |
| `Invite` | Time-limited signup tokens generated by admins |
| `ActivityLog` | Audit trail of all job/photo events for debugging and analytics |

**Key fields:**
- `Job.tvStyle` — Netflix, beach, mountains, fireplace, art, black, or off
- `Job.skyStyle` — blue-clouds, clear-blue, golden-hour, dramatic, overcast-soft, or as-is
- `Photo.bracketIndex` — position within bracket group (0=underexposed, 1=normal, 2=overexposed, etc.)
- `Photo.twilightInstructions` — optional dusk/night conversion instructions
- `User.monthlyAiCostLimit` — enforced server-side to prevent runaway costs

### AI Model Cascade

When enhancing a photo, the app tries models in priority order. If a model returns 503 (service unavailable) or 429 (rate limit), it falls through to the next:

1. **`gemini-3-pro-image-preview`** at 4K — best quality, full 20 million tokens
2. **`gemini-3.1-flash-image-preview`** at 4K — faster, still excellent quality
3. **`gemini-3.1-flash-image-preview`** at 2K — downscaled if memory/quota constraints
4. **`gemini-2.5-flash-image`** at 1K — last resort, minimal cost

Output below 4K is upscaled to 4K using Sharp (Lanczos resampling) before delivery.

### HDR Merge Strategy

Instead of pre-merging brackets with external tools (enfuse, OpenCV), the app sends **all bracket exposures** (3 or 5 images) directly to Gemini in a single multipart message with explicit HDR fusion instructions:

```
"Merge these 3 exposures into a single HDR image:
- Image 1 (underexposed): ...
- Image 2 (normal): ...
- Image 3 (overexposed): ...
Then apply color correction, fix perspective, pull windows..."
```

Gemini handles the merge AND editing in one call, eliminating intermediate files and improving consistency.

### Cost Model

| Item | Approx. Cost |
|------|--------------|
| Gemini 3 Pro (3 brackets input + 4K output, multi-turn) | $0.07 per photo |
| Manual editor (for comparison) | $0.40 per photo |
| Monthly limit enforcement | Server-side per-user quota |

Per-user monthly limits are enforced before processing starts. If a job would exceed a user's limit, processing is paused with a notification.

## API Routes

### Health & Status
- `GET /api/health` — Returns service status (database, Dropbox, Gemini, NextAuth)

### Authentication
- `POST /api/auth/[...nextauth]` — NextAuth endpoints
- `POST /api/auth/signup` — Signup endpoint (requires invite token)

### Jobs
- `POST /api/jobs/ingest` — Ingest photos from Dropbox or upload
- `POST /api/jobs/[jobId]/enhance` — Trigger enhancement for pending photos
- `POST /api/jobs/[jobId]/approve` — Bulk approve photos
- `POST /api/jobs/[jobId]/reject` — Bulk reject photos

### Photos
- `POST /api/photos/[photoId]/regenerate` — Re-enhance a single photo
- `POST /api/photos/[photoId]/approve` — Approve individual photo
- `POST /api/photos/[photoId]/reject` — Reject individual photo

### Utilities
- `GET /api/activity` — Activity log entries (paginated)
- `GET /api/dropbox/folders` — List Dropbox subfolders (for navigation)

## Performance & Limits

### Rate Limits
- Dropbox API: 300 calls per 15 minutes (app-level quota)
- Google Gemini: Varies by model and tier; fallback cascade handles transient errors
- NextAuth: Standard session limits apply

### Costs
- Gemini 3 Pro Image: ~$0.07 per photo (3 brackets + 4K output)
- Monthly limits: Enforced per user (default $50, configurable)
- Dropbox: Free tier or paid (usage-based)

### Storage
- Edited photos stored in Dropbox (user's account)
- Database stores metadata only (URLs, status, EXIF)
- Temporary upscaling work files are cleaned up after download

## Troubleshooting

### Common Issues

**"Photo enhancement timed out"**
- Gemini 3 Pro call exceeded budget or service unavailable
- Check `errorMessage` in Photo record
- Retry via UI or check model cascade configuration

**"Dropbox folder not found"**
- Verify Dropbox link is a shared folder (not file)
- Check DROPBOX_ACCESS_TOKEN has correct scopes
- Confirm photographer has Dropbox account access

**"Database connection refused"**
- Verify DATABASE_URL is correct and database is online
- Check network firewall allows outbound to Neon/Postgres
- Ensure SSL/TLS certificates are valid

**"Monthly cost limit exceeded"**
- User's `monthlyAiCostLimit` reached
- Admin must increase limit in Users table or wait for month reset
- Activity log shows which photos triggered limit

### Debug Mode

Set `DEBUG=*` environment variable to enable verbose logging:

```bash
DEBUG=* npm run dev
```

Check server logs at `/api/activity` for full event history.
